Debugging with strace, lsof, and gdb for System-Level Errors

When programs fail silently or behave unexpectedly at the system level‚Äîsuch as crashing, hanging, or failing to open files‚Äîsystem-level debugging tools become invaluable. Tools like strace, lsof, and gdb let you inspect program behavior from the kernel interaction level up to memory and symbol resolution.

üîó 2. lsof: List Open Files

Every file, socket, pipe, and device in Unix is a file descriptor. lsof shows which files a process has open.

üß™ Use Case: Check if a File or Port is in Use

lsof /path/to/file

lsof -i :8080

Shows which process is using port 8080.

üß™ Use Case: See What Files a Process Has Open

lsof -p <pid>

Helpful for debugging leaks, unclosed descriptors, or resource locks.
